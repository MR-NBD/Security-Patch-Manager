Configurare **Single Sign-On (SSO)** tra **Uyuni** e **Microsoft Entra ID** usando **SAML 2.0**, consentendo agli utenti di autenticarsi su Uyuni con le credenziali aziendali Azure AD.

L'integrazione è necessaria perché Azure AD **non espone LDAP o Kerberos** per integrazioni dirette con applicazioni on-premise come Uyuni.

# Architettura di autenticazione

Il flusso di login segue il modello **Service Provider (Uyuni) → Identity Provider (Azure AD)**:

1. L’utente apre la **Web UI di Uyuni**.
2. Uyuni reindirizza il browser verso **Azure AD** per l’autenticazione.
3. Azure AD autentica l’utente.
4. Azure invia una **SAML Assertion** a Uyuni.
5. Uyuni identifica l’utente tramite il claim **`uid` (UPN Azure AD)**.
6. Se l’utente esiste nel database Uyuni, viene effettuato il login.

⚠️ **Uyuni non supporta provisioning automatico utenti (JIT)**:  
gli utenti devono essere **creati manualmente prima del primo accesso**.

## Applicazione patch (Action scheduling)

Gli aggiornamenti vengono applicati tramite **Actions** pianificate.
Uyuni può:

- aggiornare **singolo sistema**
- aggiornare **system group**
- aggiornare **intera infrastruttura**

Tipi di azione:

- `Apply patch`
- `Update packages`
- `Dist upgrade`
- `Apply errata`
    

Le azioni possono essere:

- immediate
- pianificate
- ricorrenti
